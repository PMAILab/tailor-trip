# Task: Handle Mood Selection Interaction

**Feature Area:** Mood-Based Discovery
**Status:** Todo
**Priority:** Critical

## Description
Implement the interaction logic when a user taps on a Mood Card. This action must trigger the generation of personalized destination recommendations instantly, without forcing a user login.

## Acceptance Criteria
- [ ] Make Mood Cards tappable with clear visual feedback (e.g., ripple effect or scale change).
- [ ] Capture the selected mood state in the application state management.
- [ ] Trigger the destination fetching process immediately upon selection.
- [ ] Ensure the flow can be completed by unauthenticated/guest users.
- [ ] Show a loading state or transition gracefully while recommendations are fetched.
