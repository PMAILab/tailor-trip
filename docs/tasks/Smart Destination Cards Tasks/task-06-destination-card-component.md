# Task: Build Smart Destination Card Component

**Feature Area:** Smart Destination Cards
**Status:** Todo
**Priority:** Critical

## Description
Create a polished, reusable `DestinationCard` component that displays curated destination recommendations. The card should surface key decision-making information at a glance — destination identity, cost range, and contextual badges — while supporting multiple view states.

## Card Anatomy
- **Hero Image:** Full-width destination photo with a subtle gradient overlay for text readability.
- **Destination Name & Region:** Prominent title with a secondary region label.
- **Estimated Cost Range:** Displayed as a price range (e.g., "₹8,000 – ₹12,000").
- **Cheapest Month Indicator:** A small label highlighting the most affordable travel month.
- **Match Percentage:** Visual indicator showing how well the destination fits the user's selected mood.

## Acceptance Criteria
- [ ] Create a reusable `DestinationCard` component in `src/components/`.
- [ ] Display hero image, destination name, region, estimated cost range, and cheapest month.
- [ ] Show a match percentage badge (e.g., "94% Match").
- [ ] Ensure the card is responsive and looks great on mobile and desktop.
- [ ] Use smooth entrance animations when cards appear in the list.
- [ ] Follow the existing design system (Tailwind tokens, color palette).
